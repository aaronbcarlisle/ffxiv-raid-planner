# F6a — Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the redesigned app shell (72px Person-layer rail + top bar + 4-tab spine + minimal ⌘K palette + skip link) at a parallel `?shell=v2` route, leaving the live `GroupView` untouched.

**Architecture:** A `GroupRoute` gate reads `?shell=v2` and renders either the legacy `GroupView` (unchanged) or a new `NewShell`. `NewShell` shares `useGroupViewState()` for all tab/URL semantics and renders GroupView's tab bodies — first extracted into importable components — until F6b–F6e replace each. The 5 GroupView-coupled `HEADER_EVENTS` window-bus events become a typed `GroupActionsContext`; settings events keep their existing store bridge. New `nav.*` / `surface.nav` / `motion.nav-pill` tokens back the rail/spine.

**Tech Stack:** React 19 + TypeScript, react-router-dom v6, Zustand, Tailwind 4 (token-generated `@theme`), framer-motion, Vitest + @testing-library/react, ESLint 9 (boundaries + design-system plugins).
